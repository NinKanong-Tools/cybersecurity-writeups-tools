### What is SSLScan?
SSLScan is a security testing tool used by penetration testers, system administrators, and hackers to analyze a server’s SSL/TLS configuration.

### SSLScan – Service/Version Detection Options
| **Option**          | **What It Does**                                           |
| ------------------- | ---------------------------------------------------------- |
| `--ssl2`            | Check if **SSLv2** is enabled                              |
| `--ssl3`            | Check if **SSLv3** is enabled                              |
| `--tls10`           | Check if **TLS 1.0** is enabled                            |
| `--tls11`           | Check if **TLS 1.1** is enabled                            |
| `--tls12`           | Check if **TLS 1.2** is enabled                            |
| `--tls13`           | Check if **TLS 1.3** is enabled                            |
| `--tlsall`          | Test **all TLS versions** in one run                       |
| `--show-ciphers`    | Show supported **ciphers** for detected versions           |
| `--show-cipher-ids` | Show cipher IDs (numeric form, useful for DB/CVE mapping)  |
| `--iana-names`      | Use standard IANA cipher names (instead of OpenSSL ones)   |
| `--show-sigs`       | Show supported **signature algorithms** (RSA, ECDSA, etc.) |

### 9. Scan a range of IPs
````
for ip in $(seq 1 254); do sslscan 192.168.127.$ip; done
````

- for ip in $(seq 1 254); do ...; done:
   - This is a Bash for loop that iterates over a sequence of numbers from 1 to 254, generated by the seq 1 254 command.
   - $(seq 1 254) produces the numbers 1 2 3 ... 254, and each number is assigned to the variable ip in each iteration.
   - The loop executes the command inside the do ... done block for each value of ip.
- sslscan 192.168.127.$ip:
   - For each iteration, sslscan is run against the IP address 192.168.1.$ip (e.g., 192.168.1.1, 192.168.1.2, ..., 192.168.1.254).
- Quick network sweep to find hosts with weak SSL/TLS.

### 10. Combine with Nmap
````
nmap --script ssl-enum-ciphers -p 443 facebook.com
````
<img width="613" height="760" alt="image" src="https://github.com/user-attachments/assets/02cdad3a-9e3d-4213-b758-68166e5129bd" />

<img width="598" height="606" alt="image" src="https://github.com/user-attachments/assets/db008262-d28b-4edb-a007-364247a18325" />

- `--script ssl-enum-ciphers`:
    - The --script option invokes an NSE script, in this case, ssl-enum-ciphers.
    - The ssl-enum-ciphers script specifically enumerates supported SSL/TLS protocols (e.g., SSLv3, TLSv1.0–TLSv1.3) and cipher suites on the target server.
- `-p 443` : Specifies the port to scan, in this case, 443 (the default port for HTTPS).
- Alternative method; can be used with Nmap to enumerate ciphers as part of larger scanning.

### Why Hacker use SLScan:
Hackers (and ethical hackers/pentesters) use SSLScan because it helps them find weaknesses in how a server handles SSL/TLS encryption.
- To find outdated or weak SSL/TLS protocols
- To find weak ciphers
- To detect SSL/TLS misconfigurations
- To find known vulnerabilities
